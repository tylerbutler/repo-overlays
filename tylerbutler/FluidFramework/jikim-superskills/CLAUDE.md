# Fluid Framework

Distributed, real-time collaborative web application framework. Monorepo with ~325 packages across multiple pnpm workspaces.

## Build System

- **Package manager**: pnpm 10 (enforced via corepack, `node >=20.19.0`)
- **Build orchestrator**: `fluid-build` (custom tool from `@fluidframework/build-tools`)
- **CLI tool**: `flub` (from `@fluid-tools/build-cli`) for package management

### Common Commands

```bash
# Install
pnpm install

# Build everything
pnpm build

# Build single package (from package directory)
npm run build
# Or from root
fluid-build . --task build --packages @fluidframework/tree

# Type check only
tsc --project ./tsconfig.json

# Test
npm run test:mocha        # Mocha (node) tests
npm run test:jest          # Jest (browser) tests

# Lint & format
npm run eslint             # ESLint
biome check . --write      # Biome format + lint
npm run format             # Format via fluid-build

# Checks (run all policy/layer/version/format checks)
pnpm checks

# Generate changeset
pnpm changeset             # or: flub changeset add --releaseGroup client

# Clean
pnpm clean
```

## Code Style

- TypeScript with strict mode; target ES2022
- Dual ESM/CJS builds — every package ships both `lib/` (ESM) and `dist/` (CJS)
- Biome for formatting, ESLint for linting
- No `any` types — use `unknown` and narrow
- Prefer `?.` (optional chaining) and `??` (nullish coalescing)
- Use `async/await` over raw promises or callbacks
- Immutable by default: `const`, `readonly` where possible
- No underscore prefixes for private members

## Package Structure

### Release Groups (versioned together within each group)

- **client** (root workspace) — main packages, v2.91.0
  - `packages/` — core `@fluidframework/*` packages
  - `experimental/` — `@fluid-experimental/*`
  - `examples/` — `@fluid-example/*` (unpublished)
  - `azure/` — Azure-specific `@fluidframework/*` packages
- **routerlicious** (`server/routerlicious/`) — reference ordering service
- **gitrest** (`server/gitrest/`) — git-backed storage
- **historian** (`server/historian/`) — document history
- **build-tools** (`build-tools/`) — build infrastructure

### Package Namespaces

| Scope | Usage |
|-------|-------|
| `@fluidframework/*` | Public packages |
| `@fluid-experimental/*` | Experimental (unstable API) |
| `@fluid-internal/*` | Internal, not for external use |
| `@fluid-private/*` | Private internal |
| `@fluid-tools/*` | Developer tools |
| `fluid-framework` | Main umbrella package |

### Package Layout (typical)

```
packages/dds/tree/
├── src/                    # Source code
│   ├── index.ts            # Main entry
│   └── test/               # Tests (co-located)
├── lib/                    # ESM build output
├── dist/                   # CJS build output
├── api-extractor/          # API report configs (per tier)
├── tsconfig.json           # TypeScript config
├── eslint.config.mts       # ESLint flat config
└── package.json
```

## API Tiers

Packages expose multiple entry points with different API surface areas:

| Export path | Tier | Stability |
|-------------|------|-----------|
| `.` | public | Stable, semver-protected |
| `./beta` | beta | May change in minor versions |
| `./alpha` | alpha | May change at any time |
| `./legacy` | legacy | Deprecated, will be removed |
| `./internal` | internal | Not for external consumers |
| `./internal/test` | test | Test-only exports |

API reports are generated by API Extractor and stored in `api-extractor/` configs.

## Architecture: Layer System

Four architectural layers with enforced dependency boundaries (validated by `layer-check`):

```
Driver → Loader → Runtime → DataStore/DDS
```

- **Driver**: Storage and ordering service connectivity
- **Loader**: Container management and loading
- **Runtime**: Container runtime operations
- **DataStore**: Data stores and Distributed Data Structures (DDS)

Layer dependencies are documented in `PACKAGES.md` (auto-generated via `pnpm generate:packagesMd`). Cross-layer compatibility uses generation numbers + feature flags. See `LayerCompatibility.md`.

## Testing

- **Mocha** (v11): Primary test runner for node tests
- **Jest** (v29): Browser tests with Puppeteer (Chrome headless)
- **c8**: Code coverage
- Tests live in `src/test/` within each package
- Test variants: `:esm`, `:cjs`, `:prod` (production-mode emulation)
- Real service tests: `test:realsvc` (tinylicious or local)
- Stress tests: `test:stress`
- Benchmarks: `test:benchmark:report` using `@fluid-tools/benchmark`

## Changesets & Releases

- Uses `@changesets/cli` for version management
- All packages in the client release group are versioned together ("fixed" mode)
- Run `pnpm changeset` to generate a changeset for your PR
- Changeset files go in `.changeset/` with auto-generated filenames
- Changesets are validated in CI via `check:changesets`

## Supply Chain Security

pnpm is configured with strict security settings (see `DEV.md`):
- `minimumReleaseAge: 1440` — blocks packages published < 24 hours ago
- `blockExoticSubdeps: true` — no git/tarball transitive deps
- `strictDepBuilds: true` — explicit approval for dependency build scripts
- New build script packages must be added to `pnpm.onlyBuiltDependencies`

## Key Files

- `fluidBuild.config.cjs` — Task definitions and dependency graph
- `_buildProject.config.cjs` — Repo layout and release group configuration
- `biome.jsonc` — Biome formatter/linter rules
- `syncpack.config.cjs` — Dependency version synchronization
- `layerInfo.json` — Layer metadata for layer-check
- `PACKAGES.md` — Auto-generated package layer documentation
- `DEV.md` — Developer guidance (pinned deps, ESLint, pnpm security)
- `LayerCompatibility.md` — Layer compatibility system design
